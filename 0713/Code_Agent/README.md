# 淡江大學二手教科書媒合與面交管理平台 - 原型操作說明

本專案為「系統分析與設計」課程之原型展示（可操作切片）。此原型針對第一個核心功能切片：**賣家學生上架二手書、買家學生精準查詢、完成面交預約、交易狀態流轉、放鳥停權/人工申訴管理，以及違規商品檢舉下架**進行了高保真度的實作。

為了確保免安裝、免配置資料庫且能在任何瀏覽器中直接執行，本原型採用 **HTML5 + CSS3 + Pure JavaScript** 撰寫，並使用 `localStorage` 來模擬真實的關聯式資料庫讀寫與帳號狀態變更。

---

## 🚀 快速開始
1. 將本專案複製或下載至您的電腦。
2. 進入 `0713/Code_Agent/` 資料夾。
3. **直接按兩下 `index.html`** 即可在您的瀏覽器（Chrome, Edge, Safari, Firefox 等）中開啟並進行操作。不需要執行 `npm install`。

## 🎨 雙主題設計 (深色 / 淺色模式)
本原型在左側控制台內建了「主題切換」功能，可在任何時候一鍵切換，且系統內的所有語意色彩皆會自動調校以符合對比度規範：
- **🪐 深色模式 (Dark Theme - 系統預設)**
  - 以科技深藍與墨黑色作為基礎，搭配半透明毛玻璃擬物卡片，狀態與主按鈕以寶寶藍、霓虹紫等亮色點綴，展現現代前衛美感。
- **☀️ 淺色模式 (Light Theme)**
  - 基礎背景自動切換為舒適溫和的淺灰色背景，卡片與表格轉為高對比的純白色與深色描邊，文字顏色自適應轉為深slate藍以提供清晰的護眼閱讀體驗。
- **🚦 狀態語意色彩 (Semantics Colors)**
  - 🌴 交易成功 / 狀態正常：**翡翠綠 (Emerald Green)**
  - 🍓 違規警告 / 停權限制 / 商品檢舉：**玫瑰紅 (Rose Red)**
  - 💎 主要操作 / 預約按鈕：**寶寶藍 (Baby Blue)**

## 🔍 驗收條件操作指引
本原型已在左側內建了「模擬登入使用者」與「即時系統日誌 console」，您可以依照以下步驟驗證 5 大驗收條件：

### 驗收條件 1：搜尋精準度 100%
- **驗證步驟**：
  1. 切換使用者為 **買家A (陳大文)**，點擊第一分頁「買家精準搜尋與預約」。
  2. 在「關鍵字模糊搜尋」輸入特定老師姓名 `張教授`。觀察書籍列表，系統應 **100%精準** 過濾掉其他老師的書籍，僅顯示張教授的《系統分析與設計》。
  3. 輸入 ISBN 編號 `978986503`，列表應僅篩選出王老師的《演算法概論》（ISBN: 9789865034823）。
  4. 調整價格區間（例如：最低 300，最高 500），搜尋結果將即時依據數值過濾，將 $650 的演算法概論與 $250 的微積分排除。
  5. 點擊「重設篩選」恢復完整列表。

### 驗收條件 2：狀態鎖定防重複
- **驗證步驟**：
  1. 點擊左側控制台的 **「模擬 1 秒內重複預約衝突」** 紅色按鈕。
  2. 觀察右下角「系統運作日誌」：
     - 日誌將印出買家A (陳大文) 與買家C (黃小明) 在同一秒內同步搶購同一本《系統分析與設計》。
     - 顯示買家A領先幾毫秒，成功寫入資料庫，狀態變更為【已預約】。
     - 買家C 的請求被系統阻斷，並彈出【系統安全鎖定】提示框，告知書籍已遭預約鎖定。
  3. 點擊提示框的「確定」，觀察商場畫面上該書籍的按鈕是否已轉為 `已預約 (面交中)` 且呈現 Disabled（禁用），無法再次點擊，確保無重複交易或不當競爭。

### 驗收條件 3：上架操作極簡化與實體照必填防呆
- **驗證步驟**：
  1. 點擊上方導覽列的「賣家一頁式上架與管理」分頁。
  2. 切換左側模擬使用者為 **賣家B (林小美)**。
  3. 在左側上架表單中填寫欄位（科目名稱、老師、版本、ISBN、價格、書況、預期面交地點）。
  4. **故意不上傳圖片** 直接點擊「送出並立刻上架」，系統將跳出「上架失敗！請上傳書籍實體照片」的防呆阻斷 Toast 訊息。
  5. 在圖片上傳區點選並上傳書籍的實體照片（系統將在前端自動進行等比例壓縮防止 localStorage 溢出）。
  6. 重新點擊「送出並立刻上架」，右半部的「我的上架書籍管理」會 **立刻、無跳頁** 載入新上架的書籍與剛才壓縮過的實體照片。
  7. 切換回「買家精準搜尋與預約」分頁，可發現該新書籍已帶有實體照片並出現在商場中。整個上架流程在 1 頁內完成。

### 驗收條件 4：停權處分與人工申訴審核
- **驗證步驟**：
  1. **觸發放鳥停權**：
     - 切換使用者為 **賣家B (林小美)**，進入賣家分頁。此時可看到先前被 **買家A (陳大文)** 預約的《微積分》。
     - 對該筆預約點擊「放鳥回報」，並在跳出的詳情輸入框中輸入詳細原因（例如：`面交遲到超過30分鐘，完全聯絡不上`）後送出。買家A 將累計 1 次放鳥。
     - 切換使用者為 **買家C (黃小明)**。黃小明先前已被放鳥回報過 1 次。
     - 進入賣家分頁，對黃小明預約的書籍再次點擊「放鳥回報」並輸入放鳥原因。
     - 觀察日誌，系統會檢測黃小明累計放鳥次數達 2 次，**自動將其標記為「停權」狀態**。
  2. **申訴流程**：
     - 切換登入為被停權的 **買家C (黃小明)**。此時左側個人資料卡會顯示紅色的 `狀態：已停權`。
     - 側邊欄下方會自動跳出「帳號遭停權申訴」表單。輸入理由（例如：`當天機車突然爆胎所以沒趕上，真的很抱歉`）後點擊「送出人工申訴」。
  3. **管理員人工審核**：
     - 切換登入為 **廖語涵 (系統管理員)**，並點擊上方導覽分頁「管理後台審核中心」（非管理員切換會被攔截阻斷）。
     - 在右側「學生帳號狀態與放鳥申訴審理」清單中，會看到黃小明提交的申訴理由。
     - 管理員審閱其爆胎理由與放鳥歷史詳情後，點擊「核准 (恢復權限)」。
     - 觀察左側使用者狀態，黃小明的放鳥次數已被重設為 0，且狀態恢復正常，可再次進行買賣教科書。

### 驗收條件 5：違規商品檢舉與管理員一鍵下架
- **驗證步驟**：
  1. **發起檢舉**：
     - 切換使用者為 **買家A (陳大文)**，進入第一分頁「買家精準搜尋與預約」。
     - 在書籍列表中，找到任何非陳大文上架的書籍（例如林小美上架的《微積分》），點擊卡片下方的紅色「**⚠️檢舉**」按鈕。
     - 於確認視窗中點擊「確定」，並在隨後彈出的輸入框中填寫具體原因（例如：`此為非教科書之化妝品雜物，違反平台規範`），點選確定送出。
  2. **管理員下架審核**：
     - 切換登入為 **廖語涵 (系統管理員)**，進入「管理後台審核中心」分頁。
     - 在右下角的「檢舉違規商品審查 (非教科書之雜物)」清單中，會即時出現剛才陳大文提交的檢舉案件，並清晰顯示商品名稱與檢舉理由。
     - 管理員查核後點擊「下架違規商品」按鈕。
     - 此時該商品會立刻從後台及買家商場的前台列表中同步移除，系統日誌亦會同步載入該違規下架的變更歷程。
