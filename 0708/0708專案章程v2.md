### C. 專案章程第 2 版（Project Charter v2）

#### 1. 專案題目（Project Title）

```
基於結構化查詢的校園二手教科書媒合與面交管理平台
```

#### 2. 目標使用者（Target Users）

| 角色 | 主要任務 | 主要痛點 |
| ----- | -------- | -------- |
| 想省錢的買家學生 | 透過「科目名稱」、「開課老師」或「ISBN」精準查詢所需教科書、查看書況照片、向賣家送出交易邀約 | 現有Line社群訊息洗版較快，並且同學刊登格式有所不同，導致極難以關鍵字搜尋，只能盲目查找訊息 |
| 想清書櫃的賣家學生 | 依系統規範的欄位（科目名稱、開課老師、書籍版本、書況、價格、預期面交地點）上架二手書、管理自己的書籍狀態（上架中、已預訂、已售出） | 每次發文都要自己打字想內容，容易漏寫重要資訊，導致反覆被私訊詢問；書籍售出後也懶得刪文，進而持續收到過期詢問 |
| 平台系統管理人員（小組成員） | 檢視被檢舉的違規商品（EX：非教科書之雜物）、處理面交惡意放鳥之學生停權申訴 | 傳統Line社群缺乏管理工具，無法有效懲罰黑名單，也無法過濾垃圾訊息 |

#### 3. 問題陳述（Problem Statement）

```
對於每學期有買賣二手書需求的校內學生而言，目前依賴的Line交易社群缺乏統一格式，同學刊登的書名、版本、授課老師等資訊往往因人而異（EX：同一本資訊系統分析書名有人打英文、有人打中文縮寫），導致買家學生極難透過關鍵字精準查詢，訊息也容易被刷屏洗版。此外，由於缺乏結構化的交易狀態管理，經常發生賣家書籍已售出卻未更新或刪文，以及買賣雙方在約定面交時遭受惡意放鳥卻無法處置的情形。

本專案希望針對校園二手書的刊登與面交情境，改善資訊不對稱與搜尋效率，透過強制結構化欄位（EX：科目名稱、開課老師、ISBN）確保資料齊全且格式一致，並提供清晰的交易狀態流轉與放鳥停權機制。
本期不處理：真實線上刷卡或Line Pay金流、超商跨校物流、買賣雙方即時線上聊天室功能。
```

#### 4. 專案目標（Project Goal）

```
開發一套校園二手教科書結構化媒合管理平台，透過標準化欄位與精準篩選功能解決現有社群查詢不易的痛點，並建立透明的面交預約追蹤機制，以此提升校園二手書流轉的效率與交易誠信。
```

#### 5. 利害關係人摘要（Stakeholder Summary）

```
本專案共有6位利害關係人，以下依據權力與關注程度分為四類：

1. 高權力、高關注
    - 平台系統管理人員（小組成員）：負責維護平台、審核檢舉與黑名單申訴，是落實系統規則的關鍵角色，優先設計管理後台的檢舉審核與停權處分介面。

2. 高權力、低關注
    - 學校教務處／資訊處：不直接使用系統，但關心交易安全與個資保障，具有規則限制的潛在權力。策略上採主動宣告不涉及金流與敏感個資，避免引發校方審查。

3. 低權力、高關注
    - 想省錢的買家學生／想清書櫃的賣家學生：為系統的核心操作者，直接承受目前Line社群搜尋困難、格式混亂等痛點，是需求蒐集與訪談的首要對象。

4. 低權力、低／中關注
    - 開課授課老師：關心學生買到正確版本教材，可透過ISBN與版本欄位間接滿足其需求。
    - 校園周邊實體二手書店：屬外部利益競爭者，市場衝擊有限，僅需基本留意。
```

#### 6. 範圍內項目（In-Scope）

1. 結構化二手書上架：賣家上架時必須填寫標準欄位（科目名稱、開課老師、書籍版本、ISBN、書況、價格、預期面交地點）並上傳書籍照片。
2. 多條件精準搜尋篩選：買家可透過關鍵字（EX：科目名稱、開課老師、ISBN）進行模糊查詢，並可依價格區間進行篩選。
3. 面交邀約流程：買家選定書籍後點擊「預約購買」，系統會將書籍狀態切換為「已預約」，並自動產生面交確認單。
4. 面交進度狀態管理：提供雙方確認介面，由賣家於現場面交收現後點擊「確認完成交易」，書籍狀態正式流轉為「已售出」。
5. 違規放鳥檢舉與人工申訴審核：如若面交遭受放鳥，買賣任一方可於系統按下「放鳥回報」，經累計達2次者系統自動予以停權處分；遭停權者可提出申訴，由平台系統管理人員進行人工審核，避免誤判。

#### 7. 範圍外項目（Out-of-Scope）

1. 第三方線上金流支付系統：為了在5週內收斂，系統不經手任何金流，一律採現場面交時「現金結帳」或當面轉帳。
2. 物流寄送與店到店系統：本期僅支援校園實體面交（EX：商管大樓、圖書館前），不處理郵寄或超商代收付。
3. 即時線上聊天室：不開發像蝦皮那樣的即時通訊功能，雙方聯繫一律透過系統內建的面交時間地點約定欄位。
4. ISBN全球資料庫API串接：不串接外部圖書館資料庫來自動抓取書名，ISBN欄位在本期純作為資料庫內的比對與搜尋索引。
5. 交易爭議退款仲裁機制：面交過程中若買賣雙方發生糾紛（EX：金額爭議、物品交付爭議），系統不介入處理，僅提供放鳥次數統計、檢舉與停權申訴功能。
